 How Block Handles Work Now

  1. Hover over any paragraph, heading, code block, etc. — a ⋮⋮ grip icon appears to the left of the block
  2. Click the grip — a context menu opens with:
    - Move Up / Move Down
    - Duplicate
    - Delete
    - Collapse/Expand (headings only)
  3. The handle is hidden on the block your cursor is currently in
  4. draggable="true" is set but drag-and-drop is not actually implemented

  Current UX Problems

  1. Nearly invisible — opacity 0 by default, 0.6 on editor hover, requires precise hover on 20×20px target
  2. No grip on active block — the block you're editing has no handle, which is counterintuitive (that's the block you most likely want to move)
  3. Fake drag — says "Drag to move" but dragging does nothing
  4. Positioned off-screen on narrow windows — left: -28px can clip outside the viewport
  5. ⋮⋮ text is too small and doesn't read as a drag handle — modern editors use a 6-dot grid (⠿ or SVG)
